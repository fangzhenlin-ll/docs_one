    # 组件化开发

    组件化开发指的是：根据封装的思想，把页面上可重用的 UI 结构封装为组件，从而方便项目的开发和维护。

    每个.vue组件都有3个部分组成

    • template -> 组件的模板结构
    • script -> 组件的 JavaScript 行为
    • style -> 组件的样式

    其中，每个组件中必须包含 template 模板结构，而 script 行为和 style 样式是可选的组成部分

    <template>
   <div class="text_box">
    <h1>这是用户自定义的组件 {{username}}</h1>
    <button @click="changename">修改用户名</button>
   </div>
</template>



<script>
// 这是固定写法
    export default{
        // 定义data数据源 
        // 注意 .vue 中的data不能指向对象  规定  必须是一个函数
        data(){
           return {
            username:'张三'
           }
        },
        methods:{
            changename(){
                // 在组件中  this指的是当前组件的实例对象
                this.username='李四'
            }
        }
    }
</script>



<style>
    .text_box{
        background: pink;
    }
</style>

<!-- less写法 -->
<style lang="less">
    .text_box{
        background: pink;
        h1{
            color:red
        }
    }
</style>

        // props 是自定义属性，允许使用者为当前组件指定初始值
        props: ['init'], //只读的 不要直接修改这个值  无法设置默认值
        data() {

            }
        },
    }

    # 组件中的样式冲突 scoped

    导致组件之间样式冲突的根本原因是：
    ① 单页面应用程序中，所有组件的 DOM 结构，都是基于唯一的 index.html 页面进行呈现的
    ② 每个组件中的样式，都会影响整个 index.html 页面中的 DOM 元素

    在组件的样式中加上 scoped属性 那么这个样式就只能这个组件使用 <style lang="less" scoped>

    父组件中修改子组件里面的样式 /deep/ h5 在样式前面加上/deep/

    # 组件的生命周期

    生命周期（Life Cycle）是指一个组件从创建 -> 运行 -> 销毁的整个阶段，强调的是一个时间段。
    生命周期函数：是由 vue 框架提供的内置函数，会伴随着组件的生命周期，自动按次序执行。
    注意：生命周期强调的是时间段，生命周期函数强调的是时间点。

    创建阶段

    # beforeCreate 创建阶段的第一个生命周期函数

    组件的props data methods 尚未被创建，不可使用！

    # created 阶段 重要！

    组件的props data methods 已创建好 都处于可用的阶段 正常用来那初始化的数据请求ajax axios 请求数据
    但是组件的模板结果尚未创造

    # beforeMount 内存中编译完成 还没插入浏览器

    将要把内存中编译好的HTMl解构渲染到浏览器中，此时浏览器当前还没有组件的DOM结构

    # mounted 此时可以操作DOM结构

    将编译完的结构渲染到浏览器 此时可以操作DOM结构

    中期阶段

    # beforeUpdata 数据改变 将要渲染

    将要根据变化过后、最新的数据,重新渲染组件的模板结构 还没渲染

    # updated 渲染完成

    已经根据最新的数据完成了组件的渲染，当数据发生变化之后，为了能操作最新的DOM结构，必须把代码写在updated函数中

    销毁阶段

    # beforeDestroy 将要销毁，但是还没被销毁

    # destroyed 完成销毁

    组件在浏览器中的结构被销毁

    # 组件中的数据共享

    1. 父子关系
    • 父向子传数据：自定义属性 props 学过
    • 子向父传数据: 子组件触发this.$emit()
    1. 兄弟关系
      • ① 创建 eventBus.js 模块，并向外共享一个 Vue 的实例对象
      • ② 在数据发送方，调用 bus.$emit('事件名称', 要发送的数据) 方法触发自定义事件
      • ③ 在数据接收方，调用 bus.$on('事件名称', 事件处理函数) 方法注册一个自定义事件

    # ref 引用

    ref 用来辅助开发者在不依赖于 jQuery 的情况下，获取 DOM 元素或组件的引用。
    每个 vue 的组件实例上，都包含一个 $refs 对象，里面存储着对应的 DOM 元素或组件的引用。
    默认情况下， 组件的 $refs 指向一个空对象。

    给子组件加 ref 那么父组件就可以直接操作子组件的this

    <Left ref="comLeft"></Left>

this.$refs.comLeft.count = 0
// this.$refs.comLeft 是 Left的this

    # this.$nextTick(cb)方法 延迟到DOM加载渲染完成后执行

    组件的 $nextTick(cb) 方法，会把 cb 回调推迟到下一个 DOM 更新周期之后执行。通俗的理解是：等组件的 DOM 更新完成之后，再执行 cb 回调函数。从而能保证 cb 回调函数可以操作到最新的 DOM 元素

    # forEach some every filter reduce

    forEach 无法在中间停止(return无效)
    some 可以停止找到后不必要的循环
    sever 判断是否全选 全选则返回true 就是数组每一项都有
    filter 讲满足条件的数组找出来变为一个新的数组
    reduce 也会循环数组的每一项

    const arr=[1,2,3,4,5,6,7]
// 找到3的索引
arr.forEach((item , index)=>{
    console.log('ok1');
    if(item === 3){
        console.log(index);
        return true;
    }
})
arr.some((item , index)=>{
    console.log('ok2');
    if(item === 3){
        //在找到对应的项后  可以return true结束循环
        console.log(index);
        return true;
    }
})

const arr=[
    {id:1,state:true,name:'西瓜',price:10,count:1}
    {id:2,state:true,name:'榴莲',price:80,count:2}
    {id:3,state:true,name:'草莓',price:60,count:3}
]
arr.every(item => item.state === true)  //输出true  加入有一项不是true  则输出false


let jiage=0
arr.filter(item => item.state).forEach(item => {
   jiage = item.price * item.count
})
console.log(jiage);
// arr.filter(item => item.state).reduce((累加的结果,当前循环项)=>{
//     item.price + item.count
// },初始值)
 const aa = arr.filter(item => item.state).reduce((amt,item)=>{
  return amt += item.price * item.count
},0)
console.log(aa);
