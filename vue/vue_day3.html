    # 过滤器

    1. 要定义到 filters 节点下，本质是一个函数
    2. 在过滤器函数中，一定要有 return 值
    3. 在过滤器的形参中，可以获取到“管道符”前面待处理的那个值
    4. 如果全局过滤器和私有过滤器名字一致，此时按照“就近原则”，调用的是”私有过滤器“

    //  设置一个过滤器 将值首字母大写！ 
//  定义div  | 后面加过滤器的名

<div id="app">{{message | capi}}</div>



// 过滤器必须被定义在filters节点下  本质是函数
    const vm = new Vue({
        el: '#app',
        data: {
            message: 'hello lin'
        },
        filters: {
            capi(val) { //val 都是|前面的值
                // 过滤器中一定要有一个返回值  return
                console.log(val.charAt(0));
                const first = val.charAt(0).toUpperCase()
                const last = val.slice(1);

                return first + last
            }
        }
    })

    # 全局过滤器

    // 使用 Vue.filter() 定义全局过滤器
    Vue.filter('capi', function (str) {
      const first = str.charAt(0).toUpperCase()
      const other = str.slice(1)
      return first + other
    })

    # 侦听器 watch

    watch 侦听器允许开发者监视数据的变化，从而针对数据的变化做特定的操作。

    1. 方法格式的侦听器
      • 缺点1：无法在刚进入页面的时候，自动触发！！！
      • 缺点2：如果侦听的是一个对象，如果对象中的属性发生了变化，不会触发侦听器！！！
    2. 对象格式的侦听器
      • 好处1：可以通过 immediate 选项，让侦听器自动触发！！！
      • 好处2：可以通过 deep 选项，让侦听器深度监听对象中每个属性的变化！！！
    
const vm = new Vue({
        el: '#app',
        data: {
            username: ''
        },
        //  /所有的侦听器 都必须定义在watch下 
        watch: {
            //  本质上是函数  监视哪个数据的变化 就哪个监听的数据名当作函数名 
            // 方法格式的侦听器 
            username(newval, oldval) {
                console.log(newval, oldval);//新值在前  旧值在后
            }
            
            //  或者监听数组里面的某个值 
            'info.username' (newval) {
                console.log(newval);
            }


            //  对象格式的侦听器 
             username: {
                 //handle就是侦听器的触发函数 
                handler: function(newval, oldval) { 
                    console.log(newval, oldval);
                },
                //  // 选项的默认值是false  作用是控制侦听器是否自动 触发一次 
                immediate: true,
                 deep: true,  
                //   深度监听 
            }
        }
    })

    # 计算属性 computed

    自我理解： 当需要用到多个值得统计等操作时候，可以用计算属性得到一个新的参数 参数等于想要得计算结果

    1. ① 虽然计算属性在声明的时候被定义为方法，但是计算属性的本质是一个属性
    2. ② 计算属性会缓存计算的结果，只有计算属性依赖的数据变化时，才会重新进行运算
        // 所有的计算属性，都要定义到 computed 节点之下
    //   计算属性在定义的时候，要定义成“方法格式” 
      computed: {
        // rgb 作为一个计算属性，被定义成了方法格式，
        // 最终，在这个方法中，要返回一个生成好的 rgb(x,x,x) 的字符串 
        rgb() {
          return `rgb(${this.r}, ${this.g}, ${this.b})`
        }
      }
